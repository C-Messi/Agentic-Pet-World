# Pet Town Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Extend Agent Cat House with a persistent pixel-art Pet Town where a uniformly defined resident pet can socialize, draw fortunes, build from safe recipes, run personality showcase stalls, generate replayable experiences, and recover bounded offline events.

**Architecture:** Add versioned pet and town contracts to the shared Zod package, then implement an event-sourced town domain on the Fastify/SQLite server. Phaser renders authoritative town projections and replays approved events; React owns observation controls and drawers; model calls are limited to validated town intents and narration while deterministic rules preserve the core loop during provider failure.

**Tech Stack:** TypeScript, pnpm, Zod, Fastify, SQLite/better-sqlite3, React, Phaser 3, Vitest, Playwright, Lucide, existing OpenAI-compatible provider adapter.

**Design Reference:** `docs/superpowers/specs/2026-07-13-pet-town-design.md`

```ts
export const PET_ANIMATION_NAMES = [
  'idle', 'walk', 'sit', 'sleep', 'happy', 'curious', 'confused',
] as const;

export type PetDefinition = {
  schemaVersion: 'pet-definition.v1';
  id: string;
  displayName: string;
  source: 'player-pet' | 'resident';
  species: string;
  spriteId: string;
  palette: {
    primary: `#${string}`;
    secondary: `#${string}`;
    accent: `#${string}`;
  };
  personality: {
    curiosity: number;
    sociability: number;
    playfulness: number;
    creativity: number;
  };
  voice: { style: string; catchphrases: string[] };
  interests: string[];
  publicBio: string;
};

export type PetSpriteManifest = {
  schemaVersion: 'pet-sprite.v1';
  pixelArt: true;
  image: string;
  frame: { width: 32; height: 32 };
  columns: 4;
  rows: 7;
  anchor: { x: 0.5; y: 0.78 };
  body: { width: number; height: number; offsetX: number; offsetY: number };
  animations: Record<(typeof PET_ANIMATION_NAMES)[number], {
    frames: [number, number, number, number];
    frameRate: number;
  }>;
};
```

Zod constraints: identifiers follow `IdentifierSchema`; names are 1-24 characters; species/style/bio are bounded to 32/80/160 characters; each score is finite `0..1`; interests are unique with at most 5 entries; catchphrases are unique with at most 3 entries; colors match `^#[0-9A-Fa-f]{6}$`; image matches `^[A-Za-z0-9][A-Za-z0-9._-]*\.png$`; body dimensions are `1..32`; offsets keep the body rectangle inside the frame; each animation has exactly four unique frame indices in `0..27`; the seven required animations must all exist and unknown keys are rejected.
